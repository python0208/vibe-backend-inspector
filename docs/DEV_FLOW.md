# DEV_FLOW.md

## 开发流程

本项目按 Phase 逐步开发。

每个 Phase 必须先完成后端核心能力，再完成前端展示，最后补充测试和文档。

## Phase 0

项目骨架。

## Phase 1

项目配置模块。

## Phase 2

OpenAPI 接口发现。

## Phase 3

数据库结构发现。

## Phase 4

接口测试执行。

## Phase 5

数据库快照对比。

## Phase 6

文件监听与开发过程记录。

## Phase 7

验收报告。

## 每阶段完成标准

1. 功能可运行。
2. 前后端能打通。
3. 有基础错误处理。
4. 有最小测试。
5. README 或 docs 更新。
